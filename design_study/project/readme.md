# latticeknight — Design System\n\nPersonal brand system for **Eduardo Neto** (digital identity: *latticeknight*), extracted from the site prototype `Latticeknight.dc.html` in this project. The site presents his products (FoundMyPro, Sentinel for Sentry), his agentic-SDLC methods, and his view of AI tooling economics.\n\n**Brand idea** — *Lattice*: interconnected systems, structured complexity, ideas becoming working systems. *Knight*: integrity, responsibility, resilience. Expressed through structure, behaviour and motion — never literal symbols. Hard bans: chess/medieval imagery, hacker/Matrix aesthetics, glowing brains, floating code, meaningless particles, excessive neon.\n\nSources: the prototype in this project; brief in `uploads/claude-design-prompt.md`; imagery in `assets/`. Site will live at **eduardoneto.com**; email hello@eduardoneto.com; github.com/latticeknight.\n\n## CONTENT FUNDAMENTALS\n- Voice: direct, thoughtful, honest, technically literate, occasionally dry. Never grandiose, corporate-consultancy, or AI-influencer.\n- Banned words: revolutionising, cutting-edge, game-changing, visionary, thought leader, 10x engineer, unlocking potential, leveraging synergies.\n- First person used sparingly; prefer verbs (Building, Exploring, Testing, Operating) over "I am / I believe" declarations.\n- No emoji, ever. No exclamation marks in practice.\n- Labels/kickers are MONO UPPERCASE with wide tracking ("THE LATTICE", "WHERE AGENTS FAIL", "PRE-LAUNCH"). The wordmark "latticeknight" is always lowercase mono.\n- Honesty is a feature: unlaunched work is explained, discarded experiments stay visible, unverifiable claims are marked [VERIFY]. Never invent metrics, customers, revenue or capabilities.\n- Example (good): "Agents can produce code quickly. The engineering problem is deciding what they should be allowed to change, how their work is verified and who remains responsible."\n- Sentence case for headings ("Built in the Real World" title-case is reserved for page subtitles). Body copy uses en-dashes sparingly, UK spelling (minimise, prioritised).\n\n## VISUAL FOUNDATIONS\n- Mood: dark, architectural, calm, precise, premium, slightly mysterious. Near-black warm backgrounds (#080a0e), graphite translucent panels, warm-white type (#e9e5dc), restrained cyan (#8ec4d6) + amber (#d8a35f) accents. Cyan = system/agent/live; amber = human/caution/in-development.\n- Type: Archivo (400–700) for everything readable; IBM Plex Mono (400/500) exclusively for labels, kickers, status chips, footnotes — always uppercase with .14–.3em tracking (exception: lowercase wordmark).\n- Surfaces: translucent panels rgba(15,18,24,.78) with backdrop blur(8px) and 1px rgba(233,229,220,.08) borders; nested facet cards use a darker rgba(8,10,14,.4). No shadows anywhere — depth comes from translucency, blur and the lattice canvas behind.\n- Backgrounds: a live connected-graph canvas sits behind all content (fixed, pointer-events none). Static contexts use assets/lattice-texture.png. Radial vignettes protect text over video.\n- Radii: 3px (buttons/chips) → 5px (facets) → 6px (cards) → 8px (large sections). Small radii on purpose — architectural, not friendly-rounded.\n- Borders: warm-white alphas (.08/.12/.18); accent borders in cyan or amber alphas signal state, not decoration.\n- Motion: one grammar — contract, travel along a path, expand, resolve. Slides/travel .95s cubic-bezier(.6,.05,.25,1); content resolve .45s ease; hovers .2s; slow 4–5.4s "breathe" opacity pulses on status dots. Nothing bounces or spins. All motion respects prefers-reduced-motion.\n- Hover states: border-color shifts toward cyan and text brightens (no fills, no scale). Links: #8ec4d6 → #b7dde9.\n- Status is structural: dot + mono label (breathing dot for active), never bright badge pills. Planned = outline dot; discarded = strikethrough + reduced opacity.\n- Layout: 1060px max content column, generous negative space, CSS grid auto-fit minmax cards, flex+gap everywhere. Fixed 58px header with gradient+blur scrim.\n- Imagery: cool near-black network photography (see assets/). Portraits modest (200px), square-ish, 6px radius, 1px border. No illustration system, no hand-drawn SVG.\n\n## ICONOGRAPHY\n- There is deliberately almost none. The system uses structural glyphs instead of an icon set:\n  - status dots (6–9px circles, filled/outline, optionally breathing)\n  - 14px × 1px accent dashes as list markers\n  - unicode arrows: → (links, flow), ↑ (enter CTA), ↳ (origin backlink), ⟲ (flip affordance), — (list bullet)\n- No icon font, no emoji, no SVG icon library. If an icon is genuinely needed, prefer a mono-glyph or a dot+label; do not import an icon set without a decision.\n- No logo exists. The brand mark is typographic: "Eduardo Neto" (Archivo 600) + "latticeknight" (mono lowercase, .22em tracking, cyan). Never draw a lattice/knight logo.\n\n## INDEX\n- styles.css — global entry (imports tokens/*)\n- tokens/ — colors.css, typography.css, spacing.css, effects.css\n- assets/ — lattice-texture.png, portrait.png, hero-lattice.mp4\n- guidelines/ — foundation specimen cards (Design System tab)\n- components/core/ — Kicker, StatusDot, Button, Panel, Chip, FlipCard (+ .d.ts, .prompt.md, cards)\n- ui_kits/website/ — static recreations of all ten screens (index, systems, sdlc, workflows, automation, tooling, lab, notes, about, contact), cross-linked for click-through\n- Latticeknight.dc.html — the full interactive prototype (source of truth)\n- SKILL.md — agent skill entry point\n\n## Intentional additions\n- None. Components were extracted 1:1 from patterns already repeated in the prototype.\n\n## Caveats\n- Fonts are loaded from Google Fonts (Archivo, IBM Plex Mono) via @import — no font binaries in-repo.\n- Component preview cards are static token-based replicas (the bundle namespace tool was unavailable in this session).\n- The Website UI kit covers all ten screens as static recreations; interactive behaviour (lattice canvas, flip cards, calculator, map) lives in the DC prototype.
